# ホシドタロ — 레어 덱 시스템 (河童·座敷童子·化け猫)

오리지널 95% / 레어 덱 합산 5% 확률형 + 이미지 폴더 자동 오픈/클로즈.
타입 체크(strict) + 매니페스트 스캔 실전 테스트 + 확률 분포(10만회 4.98%) +
보안 폴백(경로 조작 → original) 검증 완료.

## 동작 원리 — "스킨" 모델

레어 덱은 **같은 78장의 다른 그림**입니다. 카드 이름·해석·AI 결론은 오리지널을
공유하고 이미지 경로만 바뀝니다. 그래서:
- **DB 시딩 불필요** — Supabase는 아무것도 안 건드립니다 (04 마이그레이션도 실행 불필요)
- AI 캐시도 그대로 재사용 (텍스트가 동일하므로 캐시키에 덱을 넣지 않음)
- 河童가 메이저 22장뿐이어도 OK — 레어 당첨 시 그 22장 풀에서만 뽑음(오히려 특별함↑)

### 자동 오픈/클로즈
빌드 때마다 `scripts/generate-deck-manifest.mjs`가 `public/tarot-images/<덱>/`을
스캔해 매니페스트를 갱신합니다:
- 이미지 **22장(minCards) 이상** → 자동 오픈 (추첨 대상 + 컬렉션 탭 활성)
- 미달/폴더 없음 → 자동 準備中
- jpg/jpeg/png/webp 인식, 잡파일(README 등)·잘못된 파일명은 무시

### 새 덱 추가 방법 (코드 수정 0줄) ← 질문 주신 부분
1. `public/tarot-images/tengu/` 폴더를 만들고 `ar00.png` ~ `ar21.png`를 넣는다
2. (선택) `src/data/decks.config.json`의 names에 `"tengu": "天狗"` 한 줄 추가
   — 안 넣으면 폴더명이 그대로 표시됨
3. 커밋 & 배포 → **끝.** 다음 빌드에서 자동으로 추첨 풀·컬렉션 탭에 등장

확률 조정도 같은 파일: `"rareChance": 0.05` → 0.03으로 바꾸면 3%.
(대안으로 Supabase Storage+설정 테이블로 배포 없이 추가하는 방법도 있지만,
이미지가 어차피 레포에 들어가 배포가 필요하므로 빌드 스캔이 가장 단순·$0입니다.)

## 파일 배치

| 파일 | 상태 | 내용 |
|---|---|---|
| `scripts/generate-deck-manifest.mjs` | 신규 | 이미지 폴더 스캔 → 매니페스트 생성 |
| `src/data/decks.config.json` | 신규 | 확률·최소장수·표시명 (사용자 편집용) |
| `src/data/deck-manifest.json` | 신규 | 스텁 — 스크립트가 자동 덮어씀 |
| `src/lib/decks.ts` | 덮어쓰기 | 매니페스트 구동. 기존 export 호환(컬렉션 페이지 무수정) |
| `src/app/api/tarot/draw/route.ts` | 덮어쓰기 | 스프레드 단위 95:5 롤, 레어 시 덱 카드로 풀 한정 |
| `src/app/api/fortune/result/route.ts` | 덮어쓰기 | deck_key 검증·이미지 경로 |
| `src/app/api/decision/result/route.ts` | 덮어쓰기 | 〃 |
| `src/app/api/collection/cards/route.ts` | 덮어쓰기 | 레어 덱 탭 = 보유 이미지 카드만(收集率 분모 자동) |
| `package.json` + 결과 페이지 2곳 | 직접 수정 | `PATCH_deck_pages.md` (①~③ 필수, ④ 배지는 선택) |

적용 순서: 파일 배치 → package.json 수정 → 로컬에서 `npm run dev` 1회
(콘솔에 `[deck-manifest] kappa(22枚)` 뜨는지 확인) → 결과 페이지 패치 → 배포.

## 설계 메모

- **롤은 스프레드(6장) 단위** — 카드별이 아니라 판 전체가 河童로 나옴 (시각 일관성 + "당첨" 체감)
- 레어 덱이 하나도 오픈 전이면 롤 자체를 건너뜀 → 지금 배포해도 무해
- 클라이언트가 deck_key를 조작해도 `resolveDeckKey`가 매니페스트에 실존하는
  이미지일 때만 인정 → 경로 조작·없는 이미지 참조 불가
- 컬렉션은 덱별로 따로 기록됨(기존 구조 그대로) — 河童 도감이 별도로 채워짐
- 공유 이미지·OG도 image_url을 그대로 쓰므로 레어 덱 그림이 자동 반영
